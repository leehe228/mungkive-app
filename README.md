# mungkive-app
2025-1 모바일프로그래밍 팀프로젝트 - 멍카이브

## 컨벤션
### 태그
[Feat]: 새로운 기능 개발
[Fix]: 오류 및 잘못된 코드 수정
[Design]: UI(composable 구성 등) 작업
[Chore]: 사소한 작업, 기타 (e.g., 주석 추가 등)
[Refactor]: 코드 전체 개선 및 수정
[Del]: 파일 삭제
[Add]: 파일 추가
[Setting]: 프로젝트 및 환경 세팅
[Network]: 서버 연결 작업 (API 통합 등)
<br><br>

### Issue
Issue 제목: [TAG] 작업명<br>
(예시: [Design] LoginView UI 개발)

Issue 템플릿:
```
### ToDo
- [ ] 할 일
```
- Issue 생성 시 #0 형식으로 Issue Number가 생성됨
- 우측 태그에 알맞은 태그 지정
<br>

### Branch
TAG/#0 (0은 Issue Number) 이름으로 branch 생성<br> 
(예시: Design/#1)
<br><br>

### Commit
commit 메시지 작성 시 아래 양식으로 작성
[TAG] #0 - 커밋 메시지 (여기서 0은 Issue Number)<br>
(예시: [Feat] #1 - 로그인 기능 구현)
<br><br>

### Pull Request
PR 제목: [TAG] #0 - 작업명 (0은 Issue Number, 작업명은 Issue 생성 시 작업명과 동일하게)

PR 템플릿:
```
### Issue
closed #0 // issue number, PR이 완료될 때 자동으로 해당 Issue를 닫겠다는 의미

### Changed
- // 주요 변경 사항 작성

### UI
- // 디자인 작업 시 디자인 첨부
- // 기능 작업 시 기능 시연 첨부


### References
- // 참고자료 (링크 등)
```
- Issue에는 "closed #0" 작성 (0은 Issue Number)
<br><br>
